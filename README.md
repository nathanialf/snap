# pokemonsnap

A work-in-progress decompilation of *Pokémon Snap* (Nintendo 64).

This repository contains **only original code, build tooling, and documentation**
written for the purpose of decompiling the game. **No ROM data, assets, or
proprietary material from Nintendo / The Pokémon Company / Creatures Inc. /
GAME FREAK is stored here.** To build, you must legally provide your own copy
of the ROM (see [BUILDING.md](docs/BUILDING.md)).

If you find any file in this repo that appears to contain ROM-derived data,
please open an issue immediately so it can be removed.

## Goals

- A **byte-for-byte matching** rebuild of the retail ROM from C and asm in
  this repository.
- A clean, reproducible build that takes a user-supplied ROM as input,
  verifies it by SHA-1, splits it once, and rebuilds an identical image.
- Incremental: every PR moves at least one function or one data section from
  asm to matched C, or improves tooling. We do not need to land 100% in one
  go; the project is designed for steady, small contributions.

## Quick start

```sh
# 1. Clone (recursively — splat and other tools are submodules)
git clone --recursive <this-repo-url> pokemonsnap
cd pokemonsnap

# 2. Place your own legally-obtained ROM at the project root, named:
#       baserom.us.z64
#    The build will SHA-1 verify it and refuse to proceed on mismatch.

# 3. Install host dependencies and the MIPS toolchain
./tools/setup.sh

# 4. Configure (splits the ROM into asm/data; populates assets/)
make setup

# 5. Build
make
```

Full instructions, including supported ROM revisions and SHA-1s, are in
[docs/BUILDING.md](docs/BUILDING.md).

## Repository layout

```
src/         decompiled C source (tracked)
include/     headers (tracked)
asm/         split asm — regenerated from your ROM, NOT tracked
  nonmatchings/  hand-written non-matching stubs (tracked)
assets/      extracted assets — regenerated from your ROM, NOT tracked
config/      splat YAML configs, linker scripts, symbol lists (tracked)
tools/       build helpers, progress reporter, IP-safety checks (tracked)
docs/        contributor and reference docs (tracked)
```

## Contributing

See [CONTRIBUTING.md](CONTRIBUTING.md). The short version:

1. Pick a function from `asm/nonmatchings/` or an unsplit overlay.
2. Write a matching C version in `src/`.
3. Run `make` — it must produce a ROM whose SHA-1 matches the original.
4. Run `./tools/check_no_rom.sh` before committing. CI runs it too.

**Never commit any file extracted from the ROM.** If your editor or build
auto-stages something under `asm/`, `assets/`, or `build/`, stop and
investigate before pushing.
